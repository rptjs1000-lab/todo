# 프로젝트 브리프

## 프로젝트명
뽀짝 투두 (Ppojjak Todo)

## 프로젝트 설명
뽀짝 투두는 귀엽고 아기자기한 캐릭터와 비주얼로 할 일 관리를 즐겁게 만들어주는 투두 웹 앱입니다. 딱딱하고 지루한 생산성 도구 대신, 오늘의 할 일을 완료할 때마다 작은 보상과 귀여운 리액션으로 동기부여를 제공합니다. 심플한 UX 위에 귀여운 감성을 얹어, 처음 쓰는 사람도 5초 안에 쓸 수 있는 앱을 지향합니다.

## 대상 사용자
- **주 타겟**: 20대 여성, 귀여운 굿즈/스티커/다이어리를 좋아하는 유저
- **부 타겟**: 기존 투두 앱이 너무 딱딱하게 느껴져 꾸준히 못 쓴 경험이 있는 누구든
- 공통 특성: 모바일 친화적, 가볍게 쓰고 싶은 니즈, 시각적 즐거움을 중시

## 핵심 기능 (4개)
1. **할 일 추가/완료/삭제** - 할 일을 입력하고, 완료 체크 시 귀여운 애니메이션(스티커 팡!) 효과 재생
2. **카테고리 태그** - 공부, 운동, 집안일 등 이모지 기반 카테고리로 할 일을 분류
3. **오늘의 달성률 뱃지** - 오늘 완료한 비율에 따라 캐릭터 표정/멘트가 바뀌는 상단 상태 위젯
4. **로컬 데이터 저장** - 로그인 없이 브라우저 localStorage에 저장, 바로 시작 가능

## 페이지 구성
1. **메인 페이지 (홈)** - 앱의 전부. 상단 달성률 위젯 + 할 일 입력 폼 + 투두 리스트. 단일 페이지 앱으로 구성, 페이지 이동 없이 모든 인터랙션 처리
2. **완료 목록 모달** - 오늘 완료한 항목을 모아보는 슬라이드업 모달. 뽀짝한 완료 스티커 표시

## 기술 스펙
- **프론트엔드**: React + TypeScript + Tailwind CSS
- **백엔드**: Next.js API Routes (MVP에서는 API 불필요, 추후 계정 기능 확장 시 사용)
- **데이터**: 브라우저 localStorage 기반 로컬 저장 (서버 DB 없음 — MVP 범위 사수)
- **애니메이션**: Framer Motion (체크 완료 시 파티클/바운스 효과)
- **아이콘/이모지**: 트위모지(Twemoji) 또는 커스텀 SVG 캐릭터

## 데이터 모델

### Todo
| 필드 | 타입 | 설명 |
|------|------|------|
| id | string | UUID, 고유 식별자 |
| text | string | 할 일 내용 |
| category | string | 카테고리 키 (예: "study", "exercise", "home") |
| completed | boolean | 완료 여부 |
| createdAt | string | 생성 일시 (ISO 8601) |
| completedAt | string \| null | 완료 일시 |

### Category
| 필드 | 타입 | 설명 |
|------|------|------|
| key | string | 카테고리 식별자 |
| label | string | 표시 이름 (예: "공부") |
| emoji | string | 대표 이모지 (예: "📚") |
| color | string | Tailwind 색상 클래스 |

> 관계: Todo는 Category의 key를 참조 (단방향). 모든 데이터는 localStorage의 단일 JSON 키에 직렬화하여 저장.

## MVP 범위
MVP는 딱 한 가지 질문으로 판단합니다: "이것 없이도 투두를 쓸 수 있는가?"

**포함 (Must Have)**
- 할 일 추가, 완료 체크, 삭제
- 완료 시 귀여운 체크 애니메이션 (핵심 감성 요소)
- 이모지 카테고리 태그 (3~5개 고정)
- 오늘의 달성률 표시 (숫자 or 캐릭터 표정)
- localStorage 저장 (새로고침해도 유지)

**제외 (Out of Scope — v2로 미룸)**
- 회원가입 / 로그인 / 클라우드 동기화
- 반복 할 일 (매일/매주)
- 날짜별 캘린더 뷰
- 푸시 알림
- 커스텀 카테고리 추가
- 다크 모드
